## 二、五大方向工作计划

### 方向 1：继续开发（产品功能补全）

**目标**：把当前「能跑」变成「能上线」。

| 优先级 | 任务 | 交付物 | 依赖 |
|--------|------|--------|------|
| P0 | 虎皮椒支付生产配置 | 可真实收款的支付闭环 | 商户号/密钥 |
| P0 | Nginx HTTPS + 域名 | https://api.tiantianpoker.com | 域名 + SSL 证书 |
| P0 | 商户后台功能补全 | 设备管理/订单导出/结算提现 | 后端 API 完善 |
| P1 | 运营后台（Admin） | 全局场馆/设备/订单/退款管理 | 新模块开发 |
| P1 | 玩家端战绩分享 | 赛后分享卡片/排行榜 | 前端组件 |
| P1 | TV Display 多语言 | 中/英切换 | i18n 资源 |
| P2 | 多桌并行支持 | 同一场馆多桌同时开赛 | 架构调整 |
| P2 | 观战模式 | 玩家可围观正在进行的赛事 | Socket 权限分层 |

**预计工期**：5-7 天

---

### 方向 2：多服务器部署（高可用架构）

**目标**：从单机首尔扩展到「首尔主服 + 美国下载/备份 + CDN 加速」。

| 服务器 | 角色 | 部署内容 | 当前状态 |
|--------|------|----------|----------|
| 首尔 43.164.130.145 | 主服 | API + Socket + DB + 支付 | ✅ 已运行 |
| 美国 43.166.240.93 | 下载/备份 | APK 下载页 + DB 备份 + 静态资源 | ❌ 未部署 |
| Cloudflare / CDN | 加速 | 官网 + APK 分发 + WSS | ❌ 未配置 |

**具体任务**：
1. 美国服务器初始化（Node.js + Nginx + PM2）
2. 部署 APK 下载服务（player-app-release.apk / tv-display-release.apk）
3. 配置每日 DB 备份脚本（首尔 → 美国）
4. 官网静态资源 CDN 化
5. WSS（WebSocket Secure）配置

**预计工期**：2-3 天

---

### 方向 3：APP 重新设计（现代化 UI）

**目标**：从「功能可用」升级到「视觉专业」。

**设计方向**：
- **风格**：深色高级感 + 扑克绿桌元素 + 金色点缀
- **参考**：PokerStars / WSOP / GGPoker 的现代扁平化设计
- **动效**：发牌动画、筹码飞行动画、All-In 震动反馈

**重设计范围**：

| APP | 重设计内容 | 优先级 |
|-----|-----------|--------|
| Player App | 启动页/登录/大厅/牌桌/结算 全 UI 重绘 | P0 |
| TV Display | 四阶段（待机/等待/对局/结算）视觉升级 | P0 |
| 商户后台 | 现代化 Dashboard（图表/卡片/响应式） | P1 |
| Admin 后台 | 全新设计 | P1 |

**技术方案**：
- 使用 Jetpack Compose Material 3 最新组件
- 自定义 PokerTable 组件（Canvas 绘制桌布/座位/筹码）
- Lottie 动画（发牌/胜利/淘汰）
- 深色模式默认，支持浅色切换

**预计工期**：4-5 天

### 方向 5：官方网站 + 品牌包装

**目标**：打造一个「一眼专业」的官网与品牌体系。

**品牌元素**：
- **Logo**：扑克牌花色（♠）+ 锦标赛奖杯元素，现代扁平风格
- **主色**：深绿（#0D5C32）+ 金色（#FFD700）+ 暗黑（#1A1A2E）
- **字体**：中文思源黑体，英文 Inter / Montserrat
- **Slogan**：「天天扑克，天天锦标赛」或「每一局，都是决赛」

**官网结构**：

```
官网（tiantianpoker.com）
├── 首页 Hero（3D 牌桌动画 + 下载按钮）
├── 产品特色（SNG 赛制/实时对战/酒吧场景）
├── 下载页（Player App / TV Display APK）
├── 商户合作（分账说明/入驻申请）
├── 规则说明（德州扑克规则 + 赛事规则）
├── 关于我们（团队/联系方式）
└── 页脚（备案/隐私政策/用户协议）
```

**技术栈**：
- 前端：Next.js 14 + Tailwind CSS + Framer Motion
- 部署：Vercel / 美国服务器静态托管
- 设计：Figma 出稿 → 前端还原

**包装清单**：
- [ ] Logo 设计（SVG/PNG 多尺寸）
- [ ] APP 图标（Player App + TV Display）
- [ ] 官网 UI 设计稿
- [ ] 宣传图（酒吧场景图/手机截图）

## 四、风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| 虎皮椒审核慢 | 支付无法上线 | 准备备用支付通道（易支付/码支付） |
| 域名备案 | 官网无法国内访问 | 使用香港/美国 CDN，或申请备案 |
| APP 重设计延期 | 整体进度滞后 | 先出核心页面，次要页面后续迭代 |
| 多服务器同步复杂 | 部署出错 | 使用 Ansible / Docker Compose 统一管理 |
| 并发性能不足 | 6 人桌卡顿 | 提前压测，必要时加 Redis 缓存 |
