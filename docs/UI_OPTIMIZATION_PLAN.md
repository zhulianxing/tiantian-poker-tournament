# 天天扑克锦标赛 — UI/UX 全面优化方案

> 目标：TV 端达到「酒吧电视直播决赛桌」水准，玩家端达到「顶级扑克手游」水准。
> 场景定义：TV 挂在酒吧墙上，观众距离 2–5 米、环境昏暗嘈杂、无人操作；
> 玩家端手机竖屏单手操作，行动有 15 秒时限，需要零误触。

---

## 一、设计原则

### TV 端（酒吧 / 远距离观看）
1. **公共牌是唯一主角**：五张公共牌必须占据画面视觉中心，尺寸拉到极限（1080p 下单张 ≥ 140×200dp，约 1/7 屏宽），3 米外一眼读牌。
2. **3 秒可读性**：任何信息（底池、轮到谁、盲注级别）瞟一眼即可获得。字号下限：底池 ≥ 32sp 等效，座位筹码 ≥ 24sp。
3. **无人值守自解释**：画面自己讲故事——阶段徽章、行动高亮、赢家庆祝，不需要解说。
4. **招揽属性**：空闲/报名状态时二维码和「扫码入座」必须大到过马路都能扫。

### 玩家端（手机 / 单手 / 限时决策）
1. **拇指热区**：所有决策操作集中在屏幕下 1/3，大拇指自然弧度可达。
2. **轮到我 = 全身都知道**：行动按钮 + 倒计时 + 震动三重提示，绝不静默超时弃牌。
3. **零误触**：弃牌/全下二次确认（已有），按钮间距 ≥ 8dp，危险动作用色块区分。
4. **信息不下钻**：对手下注、剩余人数、盲注升级倒计时全部平铺，不需要点任何按钮查看。

## 二、设计系统（双端统一 Token)

### 色彩
| Token | 值 | 用途 |
|---|---|---|
| FeltGreen 渐变 | #0B5D34 → #063A20 | 桌面台呢（TV 加径向渐变+暗角） |
| BgDark | #0D1117 | 页面底 |
| Gold | #FFD700 / GoldDark #B8860B | 品牌主色、底池、赢家 |
| CardWhite | #F5F5F0 | 牌面 |
| SuitRed | #E53935 | 红桃/方块 |
| SuitBlack | #1B1B1B | 黑桃/梅花（TV 远距离可选四色牌：方块蓝 #1E88E5、梅花绿 #43A047) |
| ActionFold | #8A94A6 灰 | 弃牌 |
| ActionCall | #2E9E5B 绿 | 跟注/过牌 |
| ActionRaise | #F0A020 橙 | 加注 |
| ActionAllIn | #D4382C 红金 | 全下 |

### 字体
- 数字（筹码/底池/盲注）用等宽粗体（如 Barlow SemiBold / Roboto Mono Bold)，牌面点数用衬线感展示体，营造赌场感。
- 双端中文为主，淘汰/全下等术语中文化：淘汰、全下、过牌、跟注、加注。

### 组件库（新建，双端共用设计）
- `PokerCard` 矢量牌组件：不再用 Unicode 花色字符（玩家端「?」bug 根因），改为 Canvas/Path 绘制四套花色 + 大号点数；支持背面纹（品牌黑桃花纹）、发牌翻转动画。
- `ChipStack` 筹码堆：按金额渲染 1–5 层筹码饼，金/红/绿三色面值。
- `SeatBadge` 座位卡：头像、昵称、筹码、状态（行动中金环脉冲/淘汰灰化/全下红金框）、庄家 D 钮。
- `PotBadge` 底池：金色胶囊 + 筹码堆图标。
- `CountdownRing` 行动倒计时环。
- `WinnerBanner` 赢家横幅。
- `ActionTicker` 操作流水条。

### 动效规范
- 快速 150ms（按钮反馈）、标准 300ms（发牌/阶段切换）、庆祝 800ms（赢家）。
- 缓动：decelerate（入场）/ accelerate（筹码飞走）。TV 盒子性能有限，禁用 blur/实时阴影，用静态渐变。

## 三、TV 端详细方案（酒吧决赛桌）

### 布局（横屏 16:9,1080p 基准）

```
┌──────────────────────────────────────────────────────────┐
│ ♠天天扑克锦标赛   盲注 L3 40/80 (⏱02:31升级)  手牌#12  剩3/6 │
│                                                          │
│      [座4]          [座5]          [座0]                  │ ← 上排3座(头像+昵称+筹码)
│       └下注80┘       └D┘           └下注160┘               │ ← 座位前下注筹码堆
│                                                          │
│        ┌────┐┌────┐┌────┐┌────┐┌────┐                    │
│        │ K♥ ││ 10♠││ 7♦ ││ 7♣ ││  背面│  ← 公共牌 HERO区   │
│        └────┘├────┘├────┘├────┘├────┘    单张140×200dp    │
│               ── 转牌圈 TURN ──                           │ ← 阶段徽章(独立一行,不压牌)
│                  💰 底池 1,240                            │ ← 底池大金徽章
│                                                          │
│      [座3]                          [座1]                 │
│       └全下┘                                              │
│              [座2 · 行动金环脉冲]                          │
│                                                          │
│ ▸ 流水: Bot1 加注到 320 · Bot2 弃牌 · Bot3 全下 580       │ ← 底部操作流水条
└──────────────────────────────────────────────────────────┘
```

### 关键改动
1. **修复座位布局 P0 bug**：废弃 Column+SpaceBetween（下排被挤没），改椭圆轨道绝对定位——6 座按角度分布在桌沿（上 3、下 3，或 2-2-2 弧线），中央区域整片让给公共牌。
2. **HERO 公共牌**：5 个卡槽常驻（未发为暗纹空槽），翻/转/河逐张滑入+翻面；阶段切换时牌列做一次 300ms 脉冲。卡片高度占屏 ~28%。
3. **阶段徽章独立成行**：不再与 POT/牌重叠（修复现有 FLOP 压徽章问题）。
4. **座位前下注筹码堆**（转播式）：当前轮下注以筹码堆+金额摆在座位与桌心之间，收池时飞入底池。
5. **行动中座位**：2dp 金环 + 呼吸脉冲 + 微提亮；淘汰座位整体 40% 透明并缩小 20%。
6. **赢家时刻（800ms)**：非赢家压暗 → 赢家座位金环爆发 → 底池筹码飞向赢家 → 顶部落下 WinnerBanner「Bot1 赢得 1,240 · 两对」（showdown 事件带 handName，用上）。
7. **摊牌 reveal**：存活者底牌在其座位旁翻开（服务端已发 contenders 底牌时需确认事件字段，若无则在 hand_result 补充）。
8. **Hand History 修复**：底池显示赢家实际赢取金额（现取结算后 pot=0 的 bug)。
9. **空闲/招揽态**：双 QR（玩家下载+桌号）放大至屏高 30%，「下一场盲注」重置为初始值（修残留 bug)，加上一场冠军榜。
10. **语言统一**：Eliminated→淘汰、All-In→全下、SHOWDOWN→摊牌。

### 顶栏信息序
品牌 → 盲注级别/数值 → 升级倒计时（新增，blind_interval 客户端推算）→ 手牌号 → 剩余人数。全部 ≥ 20sp。

## 四、玩家端详细方案（手游决策台）

### 布局（竖屏）

```
┌────────────────────┐
│ SNGT01 L2 25/50 ⏱1:20 剩4/6 │ ← 紧凑顶栏(新)
│                            │
│   [座3] [座4] [座5]        │ ← 对手座位弧(筹码+下注徽标+行动金环)
│   [座2]         [座0]      │
│                            │
│   ┌──┐┌──┐┌──┐┌──┐┌──┐   │ ← 公共牌(比现大40%)
│   └──┘└──┘└──┘└──┘└──┘   │
│   ── 翻牌圈 ──   底池 550  │ ← 阶段+底池同行
│                            │
│ ▸ Bot1 跟注 50 · Bot2 弃牌 │ ← 操作流水(新,1行滚动)
│                            │
│      ┌────┐ ┌────┐        │ ← 我的底牌(大,微倾斜3°)
│      │ K♠ │ │ Q♠ │        │
│      └────┘ └────┘        │
│  我的筹码 250   已下注 0    │
│                            │
│ ▓▓▓▓▓▓▓▓░░░░░░░ 12s       │ ← 行动倒计时条(新,融进按钮区)
│ ┌────┐┌────┐┌────┐┌────┐ │
│ │弃牌 ││跟注50││加注 ││全下 │ │ ← 拇指热区四键,主行动高亮
│ └────┘└────┘└────┘└────┘ │
└────────────────────┘
```

### 关键改动
1. **P0: 牌面花色修复**。废弃 `suitSymbol()` 字符映射，换 `PokerCard` 矢量组件——红桃/方块红色、黑桃/梅花黑色，点数加粗加大，根除此处「?」。
2. **P0: 移除调试事件名**(`hand_started` 等底部文字）,`BuildConfig.DEBUG` 才显示。
3. **行动倒计时可视化**：轮到自己时按钮区上方出现 15s 倒计时条（金→红渐变）+ 最后 3 秒震动提醒。杜绝「莫名被弃牌」。
4. **对手下注徽标**：每个对手座位下方显示本轮下注额（收池时清零）。
5. **操作流水条**：底牌上方一行滚动显示最近 2–3 条动作（昵称+动作+金额）。
6. **加注面板**：点「加注」展开底部抽屉——滑杆 + 快捷档 [½底池][底池][全下]，确认键完成操作；滑杆实时显示加注后总下注。
7. **手牌结果卡**：每手结束 2 秒浮层——赢家、赢取金额、牌型（我的胜负用金/灰区分）。
8. **淘汰时刻**：专属全屏卡——最终名次（大数字）、「继续围观 / 返回大厅」双按钮。
9. **阶段徽章**：翻牌圈/转牌圈/河牌圈中文标签随公共牌出现。
10. **顶栏重做**：桌号 · 盲注级别 · 升级倒计时 · 剩余人数，一行搞定，替代现在隐晦的「L6·320」。
11. **触感**：轮到我行动 vibrate 一次；全下确认长震。
12. **动效**：发牌 300ms 翻面、筹码赢取飞入、行动按钮 150ms 按压缩放。

### 验收标准
- **3 米测试**:TV 亮屏放 3 米外，任意时刻 3 秒内读出公共牌、底池、轮到谁。
- **单手测试**：玩家端全部决策动作拇指可达，无一次误触。
- **无感超时**：测试者被问及「为什么被弃牌」的情况归零（倒计时可见）。
- **直播感**：盲测者看 TV 画面 10 秒，能说出「这是扑克比赛直播」。

### 需要服务端配合的小改动
- `hand_result` / `showdown` 事件补充：赢家金额（修 Pot:0)、赢家牌型 `handName`、摊牌者底牌数组（若尚未包含）。
- `blind_level_up` 已带 sb/bb/level，升级倒计时由客户端按 `blind_interval` 本地推算即可，无需服务端改动。
