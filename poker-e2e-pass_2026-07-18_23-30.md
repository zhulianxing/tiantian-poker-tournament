# Poker Night E2E 完整测试通过 (2026-07-18 23:00-23:30)

## 目标
修复 Player App 操作面板关键缺陷，实现端到端完整打牌流程。

## 关键修复

### 1. `gameState.tournamentId` 遗漏（最关键）
- **问题**: `GameViewModel.joinTournament()` 设置了 `_joinResult.tournamentId` 但没有设置 `_gameState.tournamentId`
- **影响**: `performAction()` 读取 `gameState.tournamentId` 发送 `player_action`，值为空 → 服务端 `activeGames.get("")` 返回 undefined → "game not found"
- **修复**: 在 `joinTournament` 成功和 "already joined" 回退分支中都设置 `gameState.tournamentId`

### 2. `table_state` 事件缺少 `tournamentId`
- **服务端**: `poker-socket` 的 `table_state` emit 添加 `tournamentId: tournament?.id`
- **客户端**: `SocketService` 解析 `table_state` 中的 `tournamentId`

### 3. Lobby 阶段未连接 Socket
- **问题**: `TableLobbyScreen` 只做 HTTP 轮询，不连接 socket → 无法实时收到 `tournament_started` 事件
- **修复**: `TableLobbyScreen` 的 `LaunchedEffect` 中调用 `viewModel.connectSocket(tableCode)`，导航条件增加 `gameState.phase == "tournament_started"`

### 4. 满员自动开赛
- **问题**: `poker-api` 的 `join` 端点只更新 `player_count` 和广播 `seat_joined`，不检查满员
- **修复**: join 成功后检查 `player_count >= max_players`，调用 `http://localhost:3001/internal/activate`

### 5. `POKER_SOCKET_PORT` 默认值错误
- **问题**: `poker-api` 广播 `seat_joined` 到 `localhost:3011`，但 `poker-socket` 在 3001
- **修复**: 默认值从 3011 改为 3001
