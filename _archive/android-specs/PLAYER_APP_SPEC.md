# Poker Night — Android Player App 功能说明

> 酒吧玩家在座位上操作的手机端，包含登录、入桌、底牌操作、行动面板等完整牌局交互。

---

## 1. 概述

| 项目 | 值 |
|------|-----|
| 包名 | `com.pokernight.player` |
| 最低 SDK | 24 (Android 7.0) |
| 目标 SDK | 34 (Android 14) |
| 方向 | 竖屏优先（横屏可选） |
| 网络 | Wi-Fi / 5G，Socket.IO 实时通信 |
| 服务端 | `43.164.130.145:80/poker/`（首尔） |
| 推送 | FCM 可选（轮到你行动时通知） |

### 1.1 用户旅程

```
打开 App
  → 登录/注册（邮箱 + 验证码）
  → 牌桌列表 / 扫码入桌
  → 选择座位（空座可用）
  → 进入牌桌游戏
  → 看底牌 → 轮到你行动 → 下注/加注/弃牌
  → 摊牌 → 看结果 → 下一手
  → 输光筹码 = 淘汰 → 观战 or 离桌
  → 胜出 → 领取虚拟筹码（下一局 buy-in）
```

### 1.2 角色说明

玩家端以 **player（玩家）** 角色连接。服务端发送完整事件，包含该玩家的底牌（仅该玩家可见）。

**各区域详细说明：**

#### 顶栏

| 元素 | 内容 | 说明 |
|------|------|------|
| 返回按钮 | ← | 弹出确认：退出会标记 sitout |
| 桌号 | C8BBGS | 当前牌桌 |

| 盲注 | 10/20 | 小盲/大盲 |
| 盲注倒计时 | 7:24 | 升级剩余(m:ss) |
| 庄家 | 庄: P3 | 庄家玩家简称 |

#### 对手区（5 个对手座位）

- **布局**：上排 3 人 + 左右 2 人，包围公共牌
- **每个座位显示**：头像（圆形容器，固定色）→ 昵称（截断 ≤ 6 字）→ 筹码数 → 状态标签
- **当前行动者**：座位边框闪烁 + 状态标签变为 "💬行动中"
- **下注指示器**：当前轮下注额显示在座位下方小标签
- **已弃牌**：半透明显示，状态标签为 "✋"

#### 公共牌区

- **0 张**（翻牌前）：显示 5 个牌背占位 + "发牌中..."
- **3 张**（翻牌）：居中三张，两侧空位隐藏
- **4 张**（转牌）+**5 张**（河牌）：逐步补齐
- 每张牌显示花色符号 + 点数
- 底池数字：金色大字，可点击展开「主池 / 边池」详情

#### 底牌区（你的手牌）

- 显眼位置（屏幕底部向上 1/3 处）
- 两张牌并排，尺寸 60×90dp，大号点数+花色
- 由背到面的翻转动画（翻牌前 → 翻牌后保持面朝上）
- 点击底牌可放大查看（popup，含成牌概率提示）

#### 信息栏

```
筹码: 1,500          已下注: 50
保底: 1,000          本轮跟注: 50
```

- **筹码**：当前总筹码
- **已下注**：本手牌已投入的筹码
- **保底**：本局筹码（初始 buy-in，输光即淘汰）
- **本轮跟注**：到当前轮为止——跟注所需最小筹码（点击可快速跟注）

#### 行动面板

**5 个按钮：**

| 按钮 | 显示 | 触发条件 | 说明 |
|------|------|---------|------|
| 弃牌 (Fold) | ✋ | 任何非过牌状态 | 弃牌退出本局 |
| 过牌 (Check) | ✓ | 下注额为 0 | 无加注时过牌 |
| 跟注 (Call) | 跟注 N | 有下注额 | 呼出到当前下注 |
| 加注 (Raise) | N-M | 非 All-In | 最小加注额 ~ 全部筹码 |
| 全下 (All-In) | ALL | 任何状态 | 梭哈全部筹码 |

**加注交互：**
- 默认显示最小加注额
- 点击「加注」→ 展开滑块（Material 3 Slider）
- 滑块范围：最小加注额 ~ 全部筹码
- 滑块上方显示实时金额
- 有两个快捷按钮：「+ 底池」和「+ 半池」
- 点击滑块外区域收起，选中金额变为「加注 X」

**防误触：**
- 非行动轮次 → 行动面板变灰，显示「等待中...」
- 行动提交后 → 按钮禁用 500ms（防双击）
- 弃牌/ALL-IN 需二次确认（Snackbar 弹窗 3s 倒计时自动确认）

## 3. 核心数据模型

### 3.1 PlayerState（玩家本地状态）

```typescript
interface PlayerState {
  playerId: string;
  nickname: string;
  token: string;
  email: string;
  chipBalance: number;       // 总余额（跨牌局）
  currentTableCode: string | null;
  currentSeat: number | null;
}
```

### 3.2 GameTableState（来自服务端）

```typescript
interface GameTableState {
  tableCode: string;
  phase: 'waiting' | 'preflop' | 'flop' | 'turn' | 'river' | 'showdown';
  level: number;
  smallBlind: number;
  bigBlind: number;
  ante: number;
  blindCountdown: number;   // 秒
  communityCards: Card[];
  mainPot: number;
  sidePots: SidePot[];
  seats: OpponentSeat[];    // 所有座位信息
  dealerSeat: number;
}
```

### 3.3 MyPlayerView（你看到的自己——单独事件，含底牌）

```typescript
interface MyPlayerView {
  mySeat: number;
  holeCards: [Card, Card];  // 仅你可见
  myChipCount: number;
  myCurrentBet: number;
  myStatus: PlayerStatus;
  toCall: number;           // 跟注所需
  minRaise: number;         // 最小加注额
  canCheck: boolean;
  isMyTurn: boolean;
  actionDeadline: number;   // 行动截止时间戳（服务端权威）
}
```

### 3.4 PlayerAction（玩家操作——客户端→服务端）

```typescript
interface PlayerAction {
  type: 'fold' | 'check' | 'call' | 'raise' | 'allIn';
  amount?: number;          // raise/allIn 时必填
  timestamp: number;
}
```

### 3.5 OpponentSeat（对手信息——不含底牌）

```typescript
interface OpponentSeat {
  seatIndex: number;
  nickname: string;
  chipCount: number;
  currentBet: number;
  status: OpponentStatus;
  avatarColor: string;
  lastAction?: string;
}
```

## 4. Socket.IO 事件协议

### 4.1 客户端 → 服务端

| 事件 | Payload | 说明 |
|------|---------|------|
| `join_table` | `{ tableCode, role: 'player', token }` | 玩家加入牌桌 |
| `leave_table` | `{}` | 离开牌桌 |
| `sit_down` | `{ seatIndex }` | 选择座位 |
| `stand_up` | `{}` | 起立（sitout） |
| `player_action` | `PlayerAction` | 执行行动 |
| `buy_in` | `{ amount }` | 买入筹码 |
| `hand_history` | `{ tableCode, page, pageSize }` | 请求手牌历史 |

### 4.2 服务端 → 客户端（玩家端独有）

| 事件 | Payload | 说明 |
|------|---------|------|
| `my_view` | `MyPlayerView` | **您的视角**（含底牌、可操作项）— 你唯一的安全数据源 |
| `table_state` | `GameTableState` | 公共牌桌状态 |
| `opponent_action` | `{ seatIndex, action, amount }` | 对手行动（不含底牌）|
| `hand_result` | `{ youWon, winAmount, winningHand, yourHoleCards }` | 一手牌结果 |
| `countdown` | `{ secondsLeft }` | 行动倒计时同步 |
| `eliminated` | `{ rank, prize }` | 你被淘汰 |
| `tournament_end` | `{ rankings, payouts }` | 锦标赛结束 |

## 6. 错误 & 边界情况

### 6.1 断线重连

| 场景 | 行为 |
|------|------|
| **Wi-Fi/数据断开 < 30s** | Socket.IO 自动重连 → 请求 `my_view`（含当前手牌）→ 恢复画面 |
| **断开 30s-5min** | 服务端标记 sitout，自动 fold 当前手牌 → 重连后显示「已自动弃牌」告知 |
| **断开 > 5min** | 服务端标记 busted（淘汰）→ 重连后通知你已退出 |
| **App 后台** | 断开连接 + 保持本地 token 和 tableCode → 回到前台时主动重连 |
| **换网络（WiFi→4G）** | 触发新连接 → 请求 `join_table`（含 token）→ 恢复 seat → 请求 `my_view` |

### 6.2 防误触

| 场景 | 处理 |
|------|------|
| **双击行动按钮** | 提交后按钮全局禁用 500ms |
| **快速多点** | 行动提交后锁定 UI 直到服务端返回 `my_view`（更新状态）|
| **非自己回合** | 行动面板完全不可交互 |
| **筹码不足** | All-In 按钮自动高亮，raise 滑块上限锁定为总筹码 |
| **最小加注** | 加注滑块起点 = max(minRaise, toCall) |
| **弃牌确认** | 弃牌弹出「确认弃牌？底牌: K♠ A♠」（大牌时警告）|

### 6.3 其他边界

| 场景 | 处理 |
|------|------|
| **邮箱验证码 60s 内重发** | 按钮禁用 + 显示剩余秒数 |
| **验证码错误（5 次）** | 锁定 5 分钟 |
| **同时多端登录** | 服务端踢出旧 WebSocket 连接 |
| **进入已满牌桌** | 显示「满员，是否观战？」→ 以 spectator 角色进入 |
| **对手玩家断线** | 座位显示 🔴 断线标签，不弃牌（等待超时自动处理）|
| **手牌回放中退出** | 无影响（数据为只读查询）|

## 8. 推送通知（P2）

**场景：** App 在后台时，轮到你行动需要及时通知

**方案：**
- 使用 FCM (Firebase Cloud Messaging)
- 服务端在 `currentPlayer` 变化时 → 向该玩家设备推送
- App 收到推送 → 显示通知栏消息：「该你行动了！C8BBGS - 还剩 23 秒」
- 点击通知 → 打开 App → 自动进入该牌桌

**FCM 令牌管理：**
- 登录时获取 FCM token → 发送到服务端 `POST /api/v2/register-device`
- 登出时移除 token

## 10. 与 TV Display App 的关系

| 维度 | Player App | TV Display App |
|------|-----------|---------------|
| 设备 | 手机 | 电视/大屏 |
| 方向 | 竖屏自适应 | 横屏 16:9 |
| 角色 | player（含底牌） | spectator（无底牌） |
| 交互 | 主动操作（下注） | 被动展示 |
| 通信 | Socket.IO + HTTP REST | Socket.IO only |
| 推送 | FCM 可选 | 不需要 |

两个 App 共用同一套服务端 Socket 事件协议，可同时连接同一桌号。推荐玩法：手机上 Player App 操作，电视上 TV Display App 投屏显示。
