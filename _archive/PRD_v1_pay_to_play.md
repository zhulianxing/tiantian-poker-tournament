# Poker Night — 产品需求与开发文档 v1.0

## 1. 产品概念

```
酒吧投币机式德州扑克
┌──────────────────────────────────────────┐
│  用户扫码 → 下载APP → 买入参赛 →        │
│  单桌SNG赛 → 赢家得奖 → 下一局开始      │
│                                          │
│  每家酒吧 = 一个场馆                      │
│  每张桌  = 一台"投币游戏机"              │
│  每次参赛 = 投一次币                     │
└──────────────────────────────────────────┘
```

**核心模式**：像游戏厅的投篮机/赛车机——用户扫码付费加入一张牌桌，进行单桌锦标赛（SNG），赢家获得奖励。和陌生人同桌，一局结束大家各自离开，下局凑够人再开。

---

## 2. 用户角色

| 角色 | 说明 | 终端 |
|------|------|------|
| **玩家** | 酒吧客人，扫码→参赛→打牌 | 手机 APP |
| **观众** | 客人或路过的人，看牌局直播 | 电视大屏 |
| **吧台/服务员** | 管理桌子开关、查看对局 | 管理平板/后台 |
| **酒吧老板** | 查看营收、设置费率 | 管理后台 |
| **运营方（我们）** | 全局管理、分成结算 | 运营后台 |

## 3. 用户流程

### 3.1 整体流程

```
酒吧装电视 → 电视显示桌号+二维码
                    ↓
客人扫码 → 下载 Poker Night 玩家 APP
                    ↓
打开 APP → 选择进入某张桌
                    ↓
付费买入（¥X） → 等待凑齐人数
                    ↓
                    凑齐？→ SNG 开赛
                    ↓
比赛 → 赢家得奖 → 回到等待
                    ↓
客人可再买入 / 离开
```

### 3.2 电视端流程

```
通电开机
  ↓
显示该酒吧/该桌二维码（动态生成）
  ↓
显示：桌号、当前状态、当前底池
  ↓
比赛进行中：展示牌局（同现在 TV Display）
  ↓
空闲状态：展示"扫码参赛"全屏引导
  ↓
显示最近赢家滚动
```

### 3.3 玩家 APP 流程

```
首次：扫描二维码下载 APP
  ↓
打开 APP → 自动识别所在桌子（或手动输入桌号）
  ↓
看到桌子信息：当前人数、买入费、奖池
  ↓
点击"买入参赛" → 付费（微信/支付宝）
  ↓
等待凑够人数（2-6人，依桌型而定）
  ↓
SNG 开始 → 打牌
  ↓
被淘汰 → 可再买入 / 离开
  ↓
赢家 → 奖金自动入账 → 可提现
```

### 3.4 付费流程

```
用户点击"买入"
  ↓
后端创建订单（金额=买入费，含平台抽成）
  ↓
内嵌 WebView / SDK 调起微信/支付宝支付
  ↓
支付成功 → 用户加入牌桌排队
  ↓
游戏币（筹码）按费率发放到账户
  ↓
比赛结束 → 赢家收到奖金
  ↓
奖金可提现 / 留存再玩
```

## 4. 核心数据模型

```
Venue (酒吧/场馆)
├── id: string
├── name: string
├── address: string
├── ratePlan: RatePlan    // 费率方案
├── qrCodeUrl: string     // 二维码链接
├── theme: { logo, colors } // 品牌定制
└── status: active/inactive

Table (牌桌)
├── id: string            // 展示用桌号 C8BBGS
├── venueId: string
├── label: string         // "吧台1号" / "VIP卡座"
├── type: sng/cash        // 比赛/现金
├── buyIn: number         // 买入费(¥)
├── rake: number          // 抽水比例(%)
├── maxPlayers: 6
├── status: playing/waiting/offline
├── playerCount: number
└── currentPrizePool: number

Tournament (比赛/对局)
├── id: string
├── tableId: string
├── buyIn: number         // 买入费
├── totalPrize: number    // 总奖池
├── playerCount: number
├── players: Player[]
├── status: registering/started/finished
├── startedAt: datetime
└── finishedAt: datetime

Player (玩家)
├── id: string
├── nickname: string
├── phone: string
├── balance: number       // 余额 (¥)
├── createdAt: datetime
└── totalWins: number

Order (订单)
├── id: string
├── playerId: string
├── tableId: string
├── tournamentId: string
├── amount: number        // 支付金额(¥)
├── status: pending/completed/refunded
├── platformFee: number   // 平台抽成
├── venueIncome: number   // 酒吧收入
└── createdAt: datetime
```

## 5. 商业模型

### 5.1 收入流向

```
玩家支付 ¥XXX
  ├── 平台抽成 10-20%（运营方）
  ├── 酒吧抽成 5-10%（场地）
  └── 奖池 70-85%（赢家瓜分）
```

### 5.2 费率方案

| 方案 | 买入费 | 平台抽成 | 酒吧抽成 | 奖池 |
|------|--------|---------|---------|------|
| 入门 | ¥10 | 20% (¥2) | 10% (¥1) | 70% (¥7) |
| 标准 | ¥25 | 15% (¥3.75) | 10% (¥2.5) | 75% (¥18.75) |
| VIP | ¥100 | 10% (¥10) | 5% (¥5) | 85% (¥85) |

### 5.3 营收估算（单桌）

| 指标 | 值 |
|------|------|
| 假设：每晚平均对局数 | 6 局 |
| 每局人数 | 4 人 |
| 每局买入费(标准) | ¥25 × 4 = ¥100 |
| 每局平台收入 | ¥3.75 × 4 = ¥15 |
| 每桌每晚 | ¥15 × 6 = ¥90 |
| 50 家酒吧 × 2 张桌 | ¥180/天 ≈ ¥5,400/月（纯平台抽成） |

```
                 │  (Express + PM2)    │
                 │  ws://.../poker/    │
                 ├─────────────────────┤
                 │  Socket.IO (牌局)   │
                 │  REST (用户/订单)   │
                 └─────────┬───────────┘
                           │
                 ┌─────────▼──────────┐
                 │   PostgreSQL        │
                 │  (用户/订单/对局)   │
                 └────────────────────┘
```

### 6.1 MVP 功能边界（v1 不做）

| 功能 | 状态 | 原因 |
|------|------|------|
| iOS APP | ❌ | 先验证市场，Android 先跑 |
| 现金桌 | ❌ | 只做 SNG 比赛，简单闭环 |
| 跨桌比赛 | ❌ | 单桌 SNG 足够 |
| 多级盲注 | ❌ | 固定盲注结构，不做阶段升级 |
| 聊天/表情 | ❌ | 电视显示对局为主 |
| 自动补人 | ❌ | 手动拼桌，氛围更好 |
| 在线提现 | ⏳ v2 | MVP 吧台现金兑付即可 |

## 7. 开发阶段

### Phase 0 — 当前状态 ✅
- 首尔服务器：基础牌局引擎（Express + Socket.IO + SQLite）
- TV Display：牌局观看（6 座 + 公共牌 + 底池）
- 两桌测试 C8BBGS / F8R0GP

### Phase 1 — 投币机模式 MVP (1-2 周)
| 模块 | 工作量 | 关键交付 |
|------|--------|---------|
| 后端：场馆+桌子管理 | 1天 | Venue/Table CRUD API |
| 后端：付费订单系统 | 2天 | 支付对接 + 订单流水 |
| 后端：用户系统 | 1天 | 注册/登录/余额 |
| 后端：SNG 赛制适配 | 2天 | 多买家->凑齐->开赛->结束->分奖 |
| 后端：抽水结算 | 1天 | 平台/酒吧/奖池自动拆分 |
| TV Display：扫码页 | 1天 | 二维码生成 + 参赛引导 |
| Player App：基础牌局 | 3天 | 手牌/下注/操作/断线重连 |
| Player App：付费流程 | 2天 | 内嵌支付+订单 |
| Player App：扫码入场 | 1天 | 扫码识别桌子 |
| 管理后台：酒吧面板 | 2天 | 查看/管理桌子+流水 |

### Phase 2 — 完善 (2-4 周)
- 微信/支付宝支付正式对接
- 酒吧品牌定制（Logo、颜色、音乐）
- 游客模式（未注册也能观战）
- 多桌选手 TTS 播报
- 后台数据看板

### Phase 3 — 规模 (4-8 周)
- 多区域服务器
- iOS APP
- 现金桌模式
- 会员体系/积分/等级
- 跨店排行榜

## 9. 风险和关键假设

| 风险 | 缓解 |
|------|------|
| 凑不够人开赛 | 短时间自动退费 + 吧台人工吆喝 |
| 用户支付失败 | 吧台可代收现金 |
| 断线/网络差 | 超时自动 fold + 重连机制 |
| 作弊/伙牌 | 初期熟人桌不严重 + 后期回放审查 |
| 未成年人 | APP 实名 + 酒吧现场把关 |
| 法律风险 | 纯娱乐赛，非博彩；奖金固定不变相赌 |
